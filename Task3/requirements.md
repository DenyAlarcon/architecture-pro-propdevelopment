# Требования к внешним интеграциям

## Требования к безопасности

1. Все вызовы внешней платформы выполняются только через выделенный интеграционный сервис `smart-home-gateway-app`; прямой доступ мобильного приложения к API партнёра запрещён.
2. Передача данных между системами выполняется только по TLS 1.2+; доступ к API партнёра разрешён только при сервисной аутентификации OAuth 2.0 (Client Credentials).
3. В интеграцию передаются только необходимые данные для выполнения команды доступа (принцип минимизации данных).
4. Персональные данные, биометрические данные и номера автомобилей не должны попадать в логи в открытом виде.
5. Для всех операций интеграции обязателен аудит: идентификатор пользователя/сервиса, действие, результат, время, `event_id`.
6. Контракты API должны быть версионированы и валидироваться на стороне PropDevelopment.
7. Для API интеграции обязательны rate limit и защита от повторной отправки команд (idempotency key).

## Протоколы аутентификации и авторизации

1. Пользовательская аутентификация в мобильном приложении: OpenID Connect (OIDC).
2. Межсервисная аутентификация для интеграции с партнёром: OAuth 2.0 (Client Credentials).
3. Токены передаются только в заголовке `Authorization`; передача токенов в URL запрещена.
4. Авторизация на стороне PropDevelopment: RBAC + ABAC (роль, ЖК, квартира/парковка, контекст запроса).
5. Для сервисных аккаунтов действует принцип наименьших привилегий и регулярный аудит прав доступа.

## Организация взаимодействия с внешней платформой

1. Схема вызова: `Собственник -> Витрина сервисов для собственников -> tenant-core-app -> smart-home-gateway-app -> Платформа партнёра «Умный дом»`.
2. Перед вызовом внешней платформы `tenant-core-app` проверяет права доступа собственника (данные `CRM`, контекст ЖК/объекта), затем передаёт запрос в `smart-home-gateway-app`.
3. Команды в партнёрскую платформу включают: управление интеллектуальным домофоном и интеллектуальным шлагбаумом.
4. Ответы и события от партнёра (успех, отказ, ошибка) возвращаются через `smart-home-gateway-app` в `tenant-core-app`, логируются и отображаются в мобильной витрине.
